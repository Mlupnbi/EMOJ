## 二、任务清单

1. **版本与更新**：对每个 **已保存 GitHub 链接** 的模组，确认游戏内/创意工坊版本是否与 `mod_profiles.json` 的 `acceptedVersions` 一致；上游大版本更新后应使用扫描器「GitHub 一键更新」或重新粘贴 URL 扫描，并查看新生成的 `*_diff.tsv`（Added/Removed/Changed Buff）。
2. **overrides**：合并各 `*_buff_scan.json` 中与 `overrides.json` 的差异；仅对 Confidence=High 且 SuggestedPhase=Stat 的可自动合并 phase=Stat（或按项目约定）。
3. **Mixed / Unknown**：各模组扫描中的低置信条目需人工决定是否写入 overrides。
4. **StableKey**：始终使用 `ModName/BuffClassName`，勿依赖运行时 BuffId。
5. **单模组细节**：若只需处理某一模组，可同时打开该模组的 `*_cursor_prompt.txt`。
6. **GitHub REST**：`*` 列为上次扫描时记录的远程分支尖端与 `releases/latest`（匿名 API 约 60 次/h）。请在扫描器 GUI 点击「检测远程更新(API)」；「远程提示」含 ▲ 时再点「GitHub 一键更新」并查看 `*_diff.tsv`。
